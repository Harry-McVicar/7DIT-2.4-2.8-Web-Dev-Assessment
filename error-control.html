Error Control

Error Control is the process by which errors are detected and corrected in data ranging from bar codes to bank details and is hence essential to all computational systems. Throughout this concept you will learn about barcodes and checksums, human error, error controlling algorithms and parity bits.

What is error control and why is it so important?

Errors occur in digital data as a result of data corruption, human error, data decay and transmintion data corruption. Data corruption involves errors in computer data that occur during writing, reading, storage, transmition or processing that result in unintentional changes. Human errors often occur during data entry with individuals typing in incorrect information. Data decay may occur to stored data as a result of failing circuits or from bits losing their magnetic orientation or electrical charge. Transmition data corruption occurs when transmition is interupted by either interfering waves or poor environmental conditions and results in incorrect data being recieved. Noise is an unwanted high frequency disturbance or interference with the desired electrical signal. If too much noise is present, it can cause the signal of the communications line to be lost or corrupted. While it can be minimized, noise will always be present and will eventually disrupt the data transmition, even with the highest quality fibre optics cable.

Barcodes, checksums and parity bits

All barcodes are designed with a final digit to check that the barcode is correct, known as a checksum. In order to calculate the checksum every second digit in the barcode is multiplied by three and all of the new digits are added together. The checksum is the number required to round the total up to the nearest ten. Parity bits work in a 2D grid arrangement in which each row and column must have an even number of each bit (0's and 1's). If there is not an even number of each bit in both the rows and the columns, an error must have occured in the data. Packets of data are generally around 1000 bytes and include the adress they are going from and to, the packet number and the total number of packets.

Human error and error controlling algorithms

Human error often occurs during the input of data and consists of four primary types. Substitution errors occur when one values is substituted for another, transposition errors occur when the position of two values are swapped while missing and extra number errors occur when values are removed or added to the dataset. The checksum is designed to pick up on these common human errors and alert the system administrator of an incorrect barcode.

Real world implications of error control

Error control is an essential process that enables us to be confident that the correct data is stored, transmitted and recieved by detecting and correcting errors at both a bit and packet level. With applications ranging from barcodes to bank details and algorithms ranging from checksums to parity bits, it is vital to all user interfaces to maintain data integrity and reliability.
